FVSOnline is an interface to the Forest Vegetation Simulator.

Hints for first time users:
  • Use the default input database while learning, load your own data once you're comfortable using the interface. Note that there is a command under the Tools menu that lets you delete all runs and related outputs so you can easily start fresh with your own data.
  • Use default settings and options until you need to change them.

Top menu items

  • Runs: Set up and make FVS simulations; the outputs go to a database.
  • Process Outputs: Make tables, graphs and reports using the outputs from previous runs.
  • Input Database: Load (or edit) the input database.
  • Tools: Access commands to download data, update software, and delete all runs.
  • Help: Display this help.

Runs

To build and make a run, first select Stands and then add Components (using the secondary menu on the right). Stands are the simulation units defined in the input database and components are sets of FVS commands (also called keywords) that that instruct FVS to invoke options or to simulate management activities. The stands and components will be added to the Contents list on the left. The first step is selecting stands is to select the FVS variant you wish to use.

Components are accessed by first selecting component type:

  • Management for specifying management actions
  • Modifiers for specify changes in the FVS predictions
  • Outputs for FVS output controls
  • Keywords for direct access to all FVS keywords
  • Your components to access components you build or load (using Build Components).

The Time scope of a simulation is automatically set to 50 or 100 years in 5 or 10 years cycles, both depending on the variant. The parameters of time can be edited and specific years you want included in the simulation can be specified as a list of years.

Run simulations while you wait (great for under 20 stands or so), or in the background as a separate process. Use the checkboxes to specify sets of outputs without having to know the keywords needed to get the outputs you desire. You can also select alternative run scripts; these are used to implement alternative models within the FVS framework.

Build Components is used to build or upload FVS keyword sets (sometimes called .kcp files). You name them and you can use add them to simulations as you wish.

Process Outputs

To review the output, first, Load Output, select the database tables you wish to explore, then Explore Data. Make Tables and Graphs for selected variables and filter the observations by selecting stands, years, species, and other elements. If you want to directly query the output database, you can specify a Custom Query that is run on the output SQLite3 database.

Tables and graphs can be added to a report, and tables can be directly downloaded as .csv files. Reports and be built and down loaded as Microsoft Word .docx files.

Input Database

You can Replace the existing database by uploading an FVS-Ready database that is stored as a SQLite3 database (use the file suffix .db) or as a Microsoft Access database (.mdb or .accdb). The built-in training data or a blank database can be installed without uploading a file.

You can Upload and insert new rows to existing database tables. This is done by uploading a .csv file that contains column headings that are also in the existing database tables. Columns in the .csv file that are not in the existing table are ignored. Columns in the existing table that have no values in the .csv file are left null.

You can View and edit existing tables. Use the Edit mode to change values or delete rows. Make the changes and then Commit them to the database. Use the New rows mode to add new rows.
